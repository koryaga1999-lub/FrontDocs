Углубление в таблицы

Базовая структура продвинутой таблицы:

                    <table>
                        <caption>Название таблицы</caption>
                        <thead>
                            <tr>
                                <th scope="col">Заголовок 1</th>
                                <th scope="col">Заголовок 2</th>
                            </tr>
                        </thead>
                        <tbody>
                            <tr>
                                <td>Данные 1</td>
                                <td>Данные 2</td>
                            </tr>
                        </tbody>
                        <tfoot>
                            <tr>
                                <td>Итого:</td>
                                <td>Сумма</td>
                            </tr>
                        </tfoot>
                    </table>

Основные секции таблицы:

  • <thead> - заголовок таблицы (обычно содержит <th>)
  • <tbody> - основное тело таблицы (содержит данные)
  • <tfoot> - подвал таблицы (итоги, примечания)
  • <caption> - название/описание таблицы

Атрибут scope — для доступности:

                    <th scope="col"> <!-- Заголовок для всего столбца -->
                    <th scope="row"> <!-- Заголовок для всей строки -->
  • scope="col" - заголовок применяется ко всему столбцу
  • scope="row" - заголовок применяется ко всей строке
  • Упрощает навигацию для скринридеров
  • Обязательно используйте для всех заголовков

Дополнительные атрибуты для сложных таблиц:

                    <!-- 1. Присваиваем ID заголовкам -->
                    <th id="city">Город</th>
                    <th id="price">Цена</th>

                    <!-- 2. Связываем данные с заголовками -->
                    <td headers="city">Рим</td>
                    <td headers="price">€200</td>
  • id - уникальный идентификатор заголовка
  • headers - перечисление ID заголовков для ячейки
  • Используйте только для очень сложных таблиц

Правила доступности:

Всегда:

  • Используйте <caption> для описания таблицы
  • Используйте <th> для заголовков
  • Добавляйте scope к каждому <th>
  • Структурируйте с <thead>, <tbody>, <tfoot>

Никогда:

  • Не используйте таблицы для вёрстки страницы
  • Не оставляйте таблицы без заголовков
  • Не создавайте слишком сложные вложенные таблицы
  • Не полагайтесь только на визуальное расположение

Пример доступной таблицы расходов:

                    <table>
                        <caption>Расходы на путешествие по Италии, июнь 2025</caption>
                        <thead>
                            <tr>
                                <th scope="col">Категория</th>
                                <th scope="col">Город</th>
                                <th scope="col">Сумма (€)</th>
                            </tr>
                        </thead>
                        <tbody>
                            <tr>
                                <th scope="row">Транспорт</th>
                                <td>Рим</td>
                                <td>150</td>
                            </tr>
                            <tr>
                                <th scope="row">Отель</th>
                                <td>Флоренция</td>
                                <td>200</td>
                            </tr>
                        </tbody>
                        <tfoot>
                            <tr>
                                <td colspan="2"><strong>Итого:</strong></td>
                                <td><strong>350</strong></td>
                            </tr>
                        </tfoot>
                    </table>

CSS для стилизации продвинутой таблицы:

                        table {
                            border-collapse: collapse;
                            width: 100%;
                            margin: 20px 0;
                        }

                        caption {
                            font-size: 1.2em;
                            font-weight: bold;
                            padding: 10px;
                            caption-side: top;
                        }

                        thead {
                            background-color: #f2f2f2;
                        }

                        tfoot {
                            background-color: #e6f3fa;
                            font-weight: bold;
                        }

                        th, td {
                            border: 1px solid #ddd;
                            padding: 12px;
                            text-align: left;
                        }

                        th[scope="col"] {
                            background-color: #4a6fa5;
                            color: white;
                        }

                        th[scope="row"] {
                            background-color: #f8f9fa;
                        }

Подробнее в основах CSS

Главные выводы:

  1. Таблицы - только для табличных данных (не для вёрстки)
  2. Доступность - обязательна (миллионы людей используют скринридеры)
  3. <caption> + scope = основа доступности
  4. Структурируйте таблицы - это проще для стилизации и понимания
  5. Тестируйте с отключённым CSS - если непонятно, улучшайте разметку

Запомните: Хорошая таблица - это та, которую могут понять все пользователи, независимо от того, как они взаимодействуют с вашим сайтом.
