Семантика в HTML: делаем сайт понятным для всех

Что это такое?

Семантика = смысл. Семантические теги — это HTML-теги, которые не просто создают блоки, а объясняют что это за блок:

  • <header> - это шапка
  • <nav> - это меню навигации
  • <article> - это самостоятельная статья
  • <footer> - это подвал сайта

Зачем это нужно?

Простой пример:

Представьте, что вы входите в дом:

  • Семантический дом: на дверях таблички "Кухня", "Спальня", "Гостиная"
  • Несемантический дом: все комнаты без табличек, просто стены

Так же и на сайте: семантические теги — это "таблички" для браузеров, поисковиков и слабовидящих людей.

Главные теги, которые нужно запомнить:

  • <header> шапка сайта или статьи
  • <nav> главное меню навигации
  • <main> основное содержимое страницы (только один на страницу!)
  • <article> независимая статья (блог, новость)
  • <section> тематический раздел внутри статьи
  • <aside> дополнительная информация (боковая панель)
  • <footer> подвал сайта

Что получаем?

Для пользователей: удобная навигация

Для поисковиков: лучшее понимание контента (лучше SEO)

Для слабовидящих: скринридеры правильно читают страницу

Для разработчиков: чистый и понятный код

Главное правило для новичка:

Не делайте бесконечное количество дивов, это неправильно!

Вместо: <div> <div> <div> Статья </div> </div> </div>

Пишите: <section> <h2> Статья </h2> <p> Текст </p> </section>

Простая аналогия:

Сайт — это книга:

  • <header> — обложка книги
  • <nav> — оглавление
  • <main> — основная часть книги
  • <section> — глава книги
  • <footer> — последняя страница с выходными данными

Золотые правила:

  1. Один <main> на страницу — это основное содержание
  2. <article> для независимого контента (новость, пост в блоге)
  3. <nav> только для главного меню
  4. Используйте заголовки (<h1> - <h6>) внутри каждого раздела
  5. Если не знаете какой тег использовать — <div> или <span>
  6. <section> группирует связанный контент, создавая отдельную секцию

Запомните: каждый семантический тег делает ваш сайт понятнее — для людей, для роботов и для вас самих! Это основа профессиональной вёрстки.
