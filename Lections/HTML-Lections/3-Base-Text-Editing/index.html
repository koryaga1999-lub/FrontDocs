Основы редактирования текста в HTML

(Ну или как сделать текст в HTML понятным)

Зачем нужна структура текста?

Без структуры текст - это сплошная стена слов, которую никто не захочет читать.

Пример:

Стена текста (без структуры):

Всем привет Сегодня такие планы: покушать, сходить погулять Как ваши дела?

Со структурой:

Всем привет!

Сегодня такие планы:

  • покушать
  • сходить погулять

Как ваши дела?

Зачем структура:

  • Люди быстро находят нужное
  • Google лучше понимает о чём страница
  • Слабовидящие могут слушать текст по частям

Заголовки: h1 - h6

Это как в книге:

  • <h1> - название книги (как заголовок, один на страницу!)
  • <h2> - названия глав
  • <h3> - подразделы в главах

Пример книги по программированию:

<h1>Основы фронтенда</h1> <!-- Главное название -->

<h2>Что такое фронтенд?</h2> <!-- Раздел 1 -->

<h2>Как начать кодить?</h2> <!-- Раздел 2 -->

<h3>Советы разработчика</h3> <!-- Подраздел -->

Важные правила:

  1. Один <h1> на странице - как одно название у книги
  2. Не прыгай через уровни - <h1> → <h2> → <h3>, а не <h1> → <h3>
  3. Не больше 3-4 уровней - иначе слишком сложно

Абзацы: <p>

Просто оберни текст:

<p>Хумус - это вкусный соус из нута.<p>

<p>Его подают с овощами или питой.<p>

Что даёт:

  • Автоматические отступы между абзацами
  • Текст становится удобным для чтения

Списки: <ul>, <ol>, <li>

2 типа списков:

1. С точками (порядок не важен):

<ul>
<li>HTML</li>
<li>CSS</li>
<li>JS</li>
</ul>
  • HTML
  • CSS
  • JS

Для чего: ингредиенты, перечень, список дел и т.п.

2. С цифрами (порядок важен):

<ol>
<li>Откройте FrontDocs</li>
<li>Перейдите во вкладку задания</li>
<li>Начните выполнять задания!</li>
</ol>
  1. Откройте FrontDocs
  2. Перейдите во вкладку задания
  3. Начните выполнять задания!

Для чего: этапы, инструкции, шаги, рецепты и т.п.

Выделение: <em> и <strong>

Не просто "курсив" и "жирный", а со смыслом:

<em> - для акцента (как интонация в речи):

<p> Я <em> очень <em> хочу есть!

<strong> - для важного (как крик):

<p><strong> Осторожно! <strong> Горячо!

Я очень хочу есть! (с интонацией)

ОСТОРОЖНО! Горячо! (важно)

6. Специальные случаи: <b>, <i>, <u>

Когда использовать:

<i> - иностранные слова, термины:

<p> Это <i>JavaScript</i> а не Java.</p>

Это JavaScript а не Java. (иностранный термин)

<b> - выделение ключевых слов (без крика):

<p> Нам нужны: <b>нут</b>, <b>чеснок</b>, <b>оливковое масло</b>.</p>

Нам нужны: нут, чеснок, оливковое масло. (ключевые слова)

<u> - исправления, имена:

<p> Слово <u>JazaScript</u> написано неверно, правильно - JavaScript.</p>

Слово JazaScript написано неверно, правильно - JavaScript. (исправление)

ЗАПОМНИ ЭТИ 7 ТЕГОВ:

  1. <h1> — главный заголовок (один!)
  2. <h2> — разделы (может быть h2, h3, h4, h5 и h6, но обычно только h2)
  3. <p> — абзацы (бесконечное количество)
  4. <ul> — списки с точками (• HTML)
  5. <ol> — списки с цифрами (1. HTML)
  6. <em> — акцент (как интонация)
  7. <strong> — важно (как крик)
