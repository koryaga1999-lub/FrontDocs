Таблицы в HTML

Основа таблицы:

<table>

    <tr>

        <th>Заголовок 1</th> <!-- заголовок (table header) -->

        <th>Заголовок 2</th>

    </tr>

    <tr>

        <td>Данные 1</td> <!-- ячейка (table data) -->

        <td>Данные 2</td>

    </tr>

</table>

Базовая структура:

Обязательные элементы:

  • <table> - контейнер таблицы
  • <tr> - строка таблицы (может быть много)
  • <td> - ячейка с данными
  • <th> - ячейка-заголовок (жирный текст)

Добавление заголовка таблицы:

<table>

    <caption>Название таблицы</caption>

    <!-- остальная таблица -->

</table>

<caption> - описание таблицы (видно пользователям и поисковикам)

Объединение ячеек:

<td colspan="2">Объединяет 2 столбца</td>

<td rowspan="2">Объединяет 2 строки</td>

  • colspan - объединение по горизонтали
  • rowspan - объединение по вертикали

Простой рабочий пример:

<table>

    <caption>Маршрут путешествия</caption>

    <tr>

        <th>Город</th>

        <th>Дата</th>

        <th>Бюджет</th>

    </tr>

    <tr>

        <td>Рим</td>

        <td>10 июня</td>

        <td>€500</td>

    </tr>

</table>

Правила для новичков:

  1. Используйте таблицы только для табличных данных (расписания, списки, сравнения)
  2. Не используйте таблицы для вёрстки страницы — это устаревший подход
  3. Всегда добавляйте <th> для заголовков (улучшает доступность)
  4. Используйте <caption> для описания таблицы
  5. Проверяйте на мобильных — таблицы должны быть читаемы
  6. Упрощайте структуру — избегайте сложных объединений ячеек
  7. Добавляйте отступы (padding) для читаемости в CSS

Чего не нужно делать:

  1. Не верстайте страницу таблицами
  2. Не создавайте таблицы без заголовков
  3. Не делайте слишком сложные объединения ячеек
  4. Не забывайте про адаптивность на мобильных (изучим в основах CSS)
  5. Не оставляйте таблицы без стилей CSS (слишком скучно)

Когда использовать таблицы:

  1. Расписания и календари
  2. Сравнительные списки (цена, характеристики)
  3. Статистические данные
  4. Результаты соревнований
  5. Любые данные, которые логично представлять в строках и столбцах

Запомните: Таблицы — для данных, а не для дизайна. Делайте их простыми, понятными и доступными для всех пользователей.
